sharekit Module

Description

This module wraps the ShareKit 2.0 framework for iOS devices (both iPhone and iPad).

For more information about ShareKit 2.0 visit the source repository at https://github.com/ShareKit/ShareKit

Accessing the sharekit Module

To access this module from JavaScript, you would do the following:

var sharekit = require("com.0x82.sharekit");

The sharekit variable is a reference to the Module object.

Reference

This modules exposes three simple methods.

sharekit.configure({ ... })

This is required and should be the first thing you call on the module. All the configuration parameters can be found on the configuration page.

sharekit.logoutFromAll()

Forgets about all the credentials stored on the device keychain.

sharekit.share({...})

Shares an item with ShareKit. The following two arguments ARE ALWAYS REQUIRED:

  • title[string]: > a title that identifies that URL
  • view[Ti.UI.View]: > a view associated with the call to share. This could be used on the iPad to show a popover from the view that called the share action. Like the Sharekit module, passing a UIToolbar or UITabBar on this parameter will cause Sharekit to use a special effect on the action sheet.

  • sharer[string]: optional

    if you supply this key to a valid Sharer, the module will try to directly share using the sharer, instead of showing the action sheet. The names of the shares should always start with the Capital Letter (ex: Facebook, Google, ReadItLater, Delicious). The list of available Sharers are:

    Copy Mail Safari PhotoAlbum TextMessage Vkontakte Evernote Facebook FoursquareV2 Flickr GoogleReader Instapaper LinkedIn Pinboard ReadItLater Tumblr Twitter

sharing links

  • link[string]: > the valid url you want to share

sharing images

  • image[blob]: > the image blob to share

sharing texts

  • text[string]: > the text block to share

sharing files

  • file[blob]: > the file blob to share
  • filename[string]: > OPTIONAL -> the filename of the file
  • mimetype[string]: > OPTIONAL but recommended, since Titanium can't currently guess a mimetype from a file

Tips and tricks / FAQ

How to limit the services available to the user?

Copy the assets/Sharers.plist to your Resources directory, edit it using XCode, remove the services you want, and use the sharers_plist_name key on the configuration call, to specify the path to your own sharer. See the Configuration documentation for more information.

Does this module uses the new iOS5 Twitter capabilities?

Yes, as long as the application runs on an iOS5 device. If not, the module automatically reverts to a in-app strategy, like before.

Help! Facebook auth doesn't work!

Please make sure you've followed the new procedure to configure Facebook, by reading the Configuration documentation

Can I i18n/l10n this module?

Yes! Please look inside the example/strings.xml file to see a file with all the Strings available to be translated. Just follow the official Titanium Guide for internationalization and you should be ok!

https://wiki.appcelerator.org/display/guides/Internationalization
