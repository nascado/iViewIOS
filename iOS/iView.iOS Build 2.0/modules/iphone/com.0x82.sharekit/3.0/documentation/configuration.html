com.0x82.sharekit Configuration

Description

To use the ShareKit module you must configure it with the credentials of the services you want to use.

If, for example, your app only shares URLs then you probably won't need image services like Flickr. In these cases it is safe to leave an API key blank.

However, it is STRONGLY recommended that you do your best to support all services for the types of sharing you support. The core principle behind ShareKit is to leave the service choices up to the user. Thus, you should not remove any services, leaving that decision up to the user.

sharekit.configure({ ... })

This method accepts a dictionary.

General Settings

  • "myappname"[string] REQUIRED

  • "myappurl" [string] REQUIRED

  • "sharemenualphabetical_order"[boolean]: OPTIONAL. Setting this to true will show the list of share services in Alphabetical Order. Setting to false with follow the implicit order

  • "sharedwithsignature":[boolean] OPTIONAL. Append 'Shared With ' signature to email (and related forms)

Twitter

Important Twitter settings to get right:

Differences between OAuth and xAuth

There are two types of authentication provided for Twitter, OAuth and xAuth. OAuth is the default and will present a web view to log the user in. xAuth presents a native entry form but requires Twitter to add xAuth to your app (you have to request it from them).

If your app has been approved for xAuth, set twitter_consumer_use_xauth to 1.

Callback URL (important to get right for OAuth users)
  1. Open your application settings at http://dev.twitter.com/apps/
  2. 'Application Type' should be set to BROWSER (not client)
  3. 'Callback URL' should match whatever you enter in twittercallbackurl. The callback url doesn't have to be an actual existing url. The user will never get to it because ShareKit intercepts it before the user is redirected. It just needs to match.

Options for Twitter:

  • "twitterconsumerkey"[string]
  • "twitterconsumersecret"[string]
  • "twittercallbackurl"[string]
  • "twitterusexauth"[boolean] OPTIONAL

Sharers.plist

In order to customize the list of sharers to present to the user, you can copy the assets/Sharers.plist to your project Resources file. Then you should edit it with Xcode and leave only the services you want to support.

To make this module load your file (and not the builtin one), you should specifiy the following option on the configuration object:

  • "sharersplistname"[string]

Example:

// Assuming the file is on the Resources directory

sharekit.configure({
  ...
  sharers_plist_name: '/Sharers.plist',
  ...
})

Again, the old method on version < 2.0 does not work anymore.
